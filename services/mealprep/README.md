# Режим «Рецепты» — компонентный мил-преп

Новый режим в pitanie-web: подбор питания из рецептов/компонентов так, чтобы
обеды и ужины готовились батчами 2 раза в неделю, а завтраки — свежие
каждый день (быстрые, без запекания).

## Методика (ksy_profood)
- КБЖУ: Харрис-Бенедикт × коэффициент активности − 15% (похудение)
- БЖУ: белок 1,5–2 г/кг, жиры 1 г/кг, углеводы — остаток
- Тарелка: ½ овощи/клетчатка, ¼ белок, ¼ сложные углеводы; белок ≥30 г/приём

## Настройки подбора (от пользователя)
- `cook_sessions_per_week` — сколько раз в неделю готовить (1/2/3).
  Делит неделю на блоки; на блок готовится батч с учётом keeps_days.
- `breakfast_max_time` — лимит времени на завтрак (мин) → отсев небыстрых.
- `session_time_budget` / `main_max_time` — лимит времени на варочную сессию
  или на приготовление обеда/ужина.
- Поэтому у каждого рецепта проставлено `time_min`, а у компонентов — типовое
  время батч-готовки. Фильтр по времени отсекает то, что не укладывается.

## Данные
- `recipes.json` — 74 рецепта из 4 гайдов (IRNBY, KKIFOOD ×2, ksy_profood),
  размечены: cook_method, no_bake, is_quick, batch_friendly, keeps_days.
  Завтраки фильтруются по is_quick + no_bake.
- `components.json` — 39 компонентов (белки/крупы/овощи/жиры) с КБЖУ на 100 г
  и сроком хранения для батч-кука. Источник: таблица взаимозаменяемости IRNBY.

## Закупка ингредиентов — ВкусВилл
Список покупок = корзина ВкусВилл (как в режиме готовых блюд).
- Переиспользуем `services/vkusvill.py`: поиск товаров + `create_cart`.
- НО нужен РЕЖИМ СЫРЫХ ИНГРЕДИЕНТОВ (мясо, крупы, овощи в упаковках),
  а не фильтр готовых блюд (`is_ready_meal`). Каждый компонент/ингредиент →
  товар ВкусВилл → нужный вес за неделю пересчитывается в кол-во упаковок (q).
- Одна недельная корзина (сырые продукты хранятся дольше готовой еды).
- Корзина отдаётся копируемой ссылкой (VPN-воркэраунд уже есть).
